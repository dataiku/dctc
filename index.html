dctc (Dataiku Cloud Transport Client) is a tool to ease the manipulation and transfer of files across various storage types:

  • Regular filesystem
  • Hadoop HDFS
  • Amazon S3
  • Google Cloud Storage
  • FTP
  • SCP / SFTP

Download and installation

Linux, Mac OS X

The easiest way to install dctc is to get our standalone binary. Simply copy and run this from a Terminal window: 

sh -c "`curl http://downloads.dataiku.com/dctc/download-dctc`"

You can run the downloaded dctc file directly from this location or put it in your PATH.

Windows

To install dctc on Windows, download the standalone executable

You can directly run this executable. You will need to have Java installed.

Quick start

For the S3 and Google Cloud Storage protocols, you first need to add an account to dctc:

% dctc add-account s3
# Follow the instructions

Running dctc without any arguments lists the available commands.
Running dctc command --help details the usage of the command.

# List the files and folders in a S3 bucket
% dctc ls s3://my-bucket

# Synchronize incrementally a GCS path to a local file
% dctc sync gs://my-bucket/my-path target-directory

# Copy a folder from GCS to HDFS, compressing all files to .gz on the way
% dctc cp -R -c gs://my-bucket/my-path hdfs:///input-data

# Dispatch the lines of a file to 8 files on S3, compressed with gzip
% dctc dispatch input-file s3://target-bucket/target-location -f random -nf 8 -c

Features

dctc solves many real-life data management problems:
  • Fast parallel downloads, direct cloud transfers
  • Incremental syncing with the sync command
  • Smart files splitting and merging with the dispatch command
  • Remote files edition with the edit command (Linux / MacOS X only)
  • Quick peek into remote files with head and tail
dctc allows you to create command aliases. Simply add the following section in your configuration file: 
[alias]
ls = ls -G
ll = ls -lh
la = ls -a
