Commands overview

dctc provides many commands to work on files. All commands manipulate the files through URLs as defined in the protocols documentation.
To some extent, dctc commands work like their UNIX counterparts.

Running dctc with no arguments lists the available commands.
All commands accept the --help option to get a detailed summary and list of options

ls and find

The ls and find commands are used to list files and folders.

  • ls lists a single folder, or folders recursively with the -R option. It displays relative paths.
  • find lists recursively, displaying absolute addresses

cat

cat displays the content of one or several files, to the standard output.
If a GZip-compressed file (ends with .gz extension) is encountered, it is automatically uncompressed. 

% dctc cat s3://bucket/file
# Displays the content of the file

% dctc cat s3://bucket/file C:\Windows\win.ini ssh://user@hostname:file
# Displays and concatenates files, across protocols

head and tail

head and tail display the beginning (resp. end) of a file, in number of lines

cp

cp copies files and folders.

% dctc cp [OPT...] SOURCE_FILES... TARGET

Target is created as a folder if it does not exist and

  • either there are several sources
  • or the source is a folder

Compression and archiving

  • Automatically compress all output files using GZip, with the -c option
  • Automatically uncompress all GZip-compressed input files, with the -u option
  • Create a Zip archive by using a "filename.zip" target and the -a option
  • Automatically unarchive input Zip archives using the -ua option
